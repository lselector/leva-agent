## Manual Setup Tests

### A. Chrome CDP — Browser-Based Web Search & LinkedIn Feed

Chrome must run with remote debugging enabled 
and be logged into the relevant sites.

The `chrome_debug` script uses a **separate Chrome profile** 
  (`~/Library/Application Support/Google/ChromeCDP`)
so your regular Chrome session is unaffected.

**Step 1 — First-time profile setup (do once)**

```bash
./chrome_debug
# A new Chrome window opens with a fresh profile.
# In that window, log into:
#   - https://www.google.com   (for web search via CDP)
#   - https://www.linkedin.com (for LinkedIn feed)
# Close the window when done — the profile is saved.
```

**Step 2 — Verify CDP is up**

```bash
./chrome_debug        # starts CDP Chrome (no-op if already running)
curl -s http://127.0.0.1:9222/json | jq '.[].title'
# Should list open Chrome tabs
```

**Step 3 — Web search via server_auto**

```bash
curl -s -X POST http://localhost:8001/api/web/search \
  -H "Content-Type: application/json" \
  -d '{"query":"Rust async tutorial"}' | jq '.'
# Expected: array of {title, url, snippet} results
```

**Step 4 — Web fetch via server_auto**

```bash
curl -s -X POST http://localhost:8001/api/web/fetch \
  -H "Content-Type: application/json" \
  -d '{"url":"https://example.com"}' | jq '.'
# Expected: {text: "..."} with page content
```

**Step 5 — LinkedIn feed via server_auto**

```bash
curl -s -X POST http://localhost:8001/api/linkedin/feed \
  -H "Content-Type: application/json" \
  -d '{}' | jq '.'
# Expected: array of feed posts with text and author
```

**Step 6 — LinkedIn like (dry run)**

```bash
curl -s -X POST http://localhost:8001/api/linkedin/like \
  -H "Content-Type: application/json" \
  -d '{"post_index":0,"dry_run":true}' | jq '.'
# Expected: confirmation message without actually liking
```

**Step 7 — Browser screenshot**

```bash
curl -s -X POST http://localhost:8001/api/web/screenshot \
  -H "Content-Type: application/json" \
  -d '{"url":"https://example.com"}' > /tmp/screenshot.png
# Response is raw PNG bytes (not JSON)
open /tmp/screenshot.png
```

**Troubleshooting:**
- `./chrome_debug --reset` — delete the CDP profile and start fresh (re-login required)
- If CDP hangs: `pkill -f "remote-debugging-port"` then `./chrome_debug`

### B. Gmail — OAuth2 Credentials Setup

The code reads from `credentials/gmail_credentials.json` (OAuth client) and
`credentials/gmail_token.json` (saved token). There is no `gmail-auth` binary yet,
so the token must be obtained via the Google OAuth Playground.

**Step 1 — Create OAuth credentials in Google Cloud Console**

1. Go to [console.cloud.google.com](https://console.cloud.google.com) → APIs & Services → Credentials
2. Create a project (or use existing)
3. Enable the **Gmail API** (APIs & Services → Library → search "Gmail API")
4. Create credentials → OAuth 2.0 Client ID → Application type: **Desktop app**
5. Download the JSON → save as `credentials/gmail_credentials.json`

The file must have this shape:
```json
{
  "installed": {
    "client_id": "YOUR_CLIENT_ID.apps.googleusercontent.com",
    "client_secret": "YOUR_CLIENT_SECRET",
    "token_uri": "https://oauth2.googleapis.com/token",
    ...
  }
}
```

**Step 2 — Get a token via Google OAuth Playground**

1. Open [oauth2.googleapis.com/playground](https://developers.google.com/oauthplayground)
2. Click the gear icon → check "Use your own OAuth credentials"
3. Enter your `client_id` and `client_secret` from step 1
4. In step 1 of the playground, paste these scopes and click "Authorize":
   ```
   https://www.googleapis.com/auth/gmail.readonly
   https://www.googleapis.com/auth/gmail.send
   https://www.googleapis.com/auth/gmail.compose
   ```
5. Sign in with your Google account and allow access
6. In step 2, click "Exchange authorization code for tokens"
7. Copy the `access_token` and `refresh_token` from the response

**Step 3 — Save the token**

```bash
mkdir -p credentials
cat > credentials/gmail_token.json << 'EOF'
{
  "access_token": "PASTE_ACCESS_TOKEN_HERE",
  "refresh_token": "PASTE_REFRESH_TOKEN_HERE",
  "expires_in": 3600
}
EOF
```

**Step 4 — Verify Gmail access**

```bash
# Check inbox
curl -s -X POST http://localhost:8001/api/gmail/inbox \
  -H "Content-Type: application/json" \
  -d '{"max_results":5}' | jq '.'
# Expected: array of emails with id, subject, from, snippet

# List labels
curl -s http://localhost:8001/api/gmail/labels | jq '.'

# Search emails
curl -s -X POST http://localhost:8001/api/gmail/search \
  -H "Content-Type: application/json" \
  -d '{"query":"is:unread","max_results":5}' | jq '.'
```

**Step 5 — Test Gmail via chat**

```bash
curl -s -X POST http://localhost:8000/api/chat \
  -H "Content-Type: application/json" \
  -d '{"message":"Check my email inbox","session_id":"gmail_test","stream":false}' | jq '.'

curl -s -X POST http://localhost:8000/api/chat \
  -H "Content-Type: application/json" \
  -d '{"message":"Find unread emails","session_id":"gmail_test","stream":false}' | jq '.'

# Clean up
rm -f memory/sessions/gmail_test.json
```

**Note:** The token auto-refreshes using `refresh_token` on each request. If you get
an auth error, repeat steps 2–3 to get a fresh token.
